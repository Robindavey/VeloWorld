Title: VeloVerse — Checkout Notes
Date: 2026-04-08

Summary
-------
Today focused on stabilizing social features, route processing reliability, and frontend API resilience for LAN/HTTPS usage. The major outcomes were:
- Route uploads now have retry support and local fallback worker startup support.
- Social flows now support incoming and outgoing request tracking, cancel sent requests, and unfollow.
- User search was rebuilt to avoid database crashes and now supports responsive recommendations in the UI.
- Frontend pages were hardened to handle text/non-JSON backend responses gracefully (no JSON token parse crashes in UI).

Current State
-------------
- Backend API is running and healthy behind the frontend HTTPS proxy (`/api/*`).
- Follow request flow works end-to-end:
  - sender can request follow,
  - recipient can see incoming pending requests,
  - sender can track outgoing pending requests,
  - sender can cancel pending outgoing requests,
  - users can unfollow accepted relationships.
- Search endpoint (`/social/users/search`) returns ranked relationship-aware results and no longer returns persistent 500 for normal and email-like queries.
- Dashboard search now includes a live recommendation popup that updates as the user types.

What Changed Today
------------------
Backend
- Added social schema bootstrap on startup (for legacy DB volumes missing social tables).
- Added route retry endpoint for stalled processing:
  - `POST /routes/{id}/retry`
- Added outgoing follow request listing endpoint:
  - `GET /social/follows/requests/outgoing`
- Added cancel outgoing follow request endpoint:
  - `DELETE /social/follows/requests/{id}`
- Added unfollow endpoint:
  - `DELETE /social/follows/{id}`
- Updated follow request creation logic to avoid conflicting pending states and prevent accepted relationships from being overwritten.
- Reworked social user search SQL to relationship-aware ranking without extension-dependent fuzzy functions that were causing runtime DB errors.

Frontend
- Added route delete actions with confirmation in upload and dashboard views.
- Added safer API response parsing helpers to avoid `Unexpected token ... is not valid JSON` UI failures.
- Added incoming and outgoing follow request panels in relevant dashboard flows.
- Added cancel request and unfollow actions in search and outgoing request lists.
- Removed separate follower-search area and prioritized follow relationships in primary user search results.
- Added recommendation dropdown that reacts to user search input and supports click-to-search.

Scripts / Runtime
- `scripts/manage.sh` enhanced local fallback mode to start/stop/status a local pipeline worker process, so queued uploads can continue processing outside Docker-first paths.
