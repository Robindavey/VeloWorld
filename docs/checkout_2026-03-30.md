How to run locally (short)
--------------------------
1. Start full stack (build + compose) and frontend HTTPS server (uses certs in project root):

   ```bash
   chmod +x scripts/manage.sh
   ./scripts/manage.sh start
   ```

   - If `frontend/localhost.pem` and `frontend/localhost-key.pem` exist the script serves HTTPS on https://localhost:8443; otherwise it falls back to http://localhost:3000.

2. Open demo: https://localhost:8443/demo.html
   - Ensure a valid `veloworld_token` is present in `localStorage` (login via API at http://localhost:8080 or paste token in DevTools).

3. Upload a GPX/FIT/TCX route via the web UI to enqueue a processing job or push a job into Redis for testing.

4. Verify pipeline output:
   - Redis key: `route_render:<route_id>` should contain JSON with `distance_m`, `elevation_gain_m`, and `profile_points` (objects with `distance_m`, `elevation_m`, optional `lat`/`lon`).
   - Backend `GetRouteRenderData` will return cached render data when key exists.

Web Bluetooth notes
-------------------
- Modern browsers require a secure context (HTTPS) for Web Bluetooth. For local dev the options are:
  - Serve the demo from `localhost` over HTTPS using a locally-trusted cert (mkcert). `frontend/serve_https.py` + `frontend/README_https.md` provide instructions.
  - Launch Chrome with the `--unsafely-treat-insecure-origin-as-secure` flag (not recommended long-term).
- The demo UI (`frontend/demo.html` and the dedicated demo page) includes a warning banner when not in a secure context.
